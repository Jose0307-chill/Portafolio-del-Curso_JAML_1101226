# Análisis de Entradas, Salidas y Procesos



#### Desafío #1



Entradas: La entrada es el número que el programa solicita al usuario.



Proceso: El programa evalúa si el número es menor, mayor o igual a cero.



Salidas: El programa muestra un mensaje al usuario como respuesta al dato ingresado, luego de evaluar las condiciones, indica si es positivo, negativo o neutro.



#### Desafío #2



Entradas: El usuario le indica al programa un año.



Proceso: El programa evalúa si el año es divisible entre 4 y es divisible entre 100 o es divisible entre 400.



Salidas: El programa indica si el año es bisiesto o no después de realizar el proceso.



#### Desafío #3



Entradas: El usuario le indica al programa su salario mensual, asimismo, le indica si tiene o no una multa en su pago.



Proceso: El programa primero evalúa el salario indicado por el usuario, luego evalúa si el usuario tiene multa o no. En base a estos datos, realiza el proceso.



Salidas: Luego de realizar el proceso, el programa le indica al usuario la cantidad que debe pagar según su salario y si tiene multa o no.



#### Desafío #4



Entradas: El usuario indica la cantidad de horas parqueado y la cantidad de dinero con la que pagará.



Proceso: El programa evalúa tres condiciones: Si el pago es menor al monto. Si el pago es igual al monto. Si el pago es mayor al monto, en este último, el programa realiza otro procedimiento de cálculo.



Salidas: El programa inicia mostrando al usuario la cantidad que debe pagar, si paga menos al monto indicado; el programa dará error, si paga lo mismo que el monto indicado; el programa indicará que no se requiere cambio, si paga más que el monto indicado; el programa mostrará la cantidad de su cambio y la cantidad de billetes.
